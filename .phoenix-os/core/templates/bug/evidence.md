# Bug Fix Evidence Template

This template defines the structured format for documenting bug fix implementation evidence.

## Template Variables

- `{issue-number}`: Bug issue number
- `{bug-title}`: Bug issue title
- `{implementation-date}`: Implementation timestamp
- `{files-changed}`: List of modified files with changes
- `{test-cases}`: List of test cases added/updated
- `{quality-gate-results}`: Quality gate check results
- `{blockers}`: Any blockers encountered during implementation

## Formatting Rules

### Files Changed
- List all modified files
- Include line ranges for changes
- Specify change type (added, modified, removed)
- Provide clear rationale for each change

### Test Cases
- List all test cases added or modified
- Specify test file locations
- Describe what each test verifies
- Include test results (pass/fail)

### Quality Gate Results
- Provide detailed output from each quality gate
- Use checkboxes for gate checklist
- Include pass/fail counts and percentages
- Show relevant error messages if gates fail

### Blockers
- List both active and resolved blockers
- Describe impact of each blocker
- Provide resolution steps or status
- Prioritize active blockers

### Verification Steps
- Confirm bug was reproducible before fix
- Verify bug no longer occurs after fix
- Check for regressions in related functionality
- Validate all quality gates pass

## Implementation Notes

1. **Variable Substitution**: Agent should replace all `{variable}` placeholders with actual values
2. **Status Tracking**: Update status as implementation progresses
3. **Evidence Collection**: Capture actual test output, not just summaries
4. **Completeness**: Document all changes, even minor ones
5. **Traceability**: Link changes back to RCA recommendations
