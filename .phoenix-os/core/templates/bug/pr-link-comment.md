# PR Link Comment Template

This template defines the format for posting a PR link comment to the bug issue after PR creation.

## Template Variables

- `{issue-number}`: Bug issue number
- `{pr-number}`: Pull request number
- `{pr-url}`: Full URL to the pull request
- `{pr-title}`: Pull request title
- `{branch-name}`: Branch name for the fix
- `{files-changed-count}`: Number of files changed
- `{tests-added-count}`: Number of tests added

## Output Format

```markdown
## Fix Implementation Started

A pull request has been created to address this bug:

**PR**: [{pr-title}]({pr-url})
**Branch**: `{branch-name}`

### Summary

- Files changed: {files-changed-count}
- Tests added: {tests-added-count}
- Quality gates: All passing

### Next Steps

1. Review the PR changes
2. Wait for CI/CD checks to complete
3. Approve and merge when ready

The issue will be automatically closed when the PR is merged.

---
Generated by Phoenix OS Bug Fix Workflow
```

## Formatting Rules

### PR Reference
- Use markdown link format for clickable PR reference
- Include branch name for tracking

### Summary Stats
- Keep numbers simple and clear
- Show quality gate status

### Next Steps
- Number steps for clarity
- Keep actionable and concise

## Implementation Notes

1. **Linking**: Ensure PR URL is clickable
2. **Auto-close**: Remind that issue will auto-close on merge
3. **Brevity**: Keep comment short and focused
4. **Status Update**: This serves as a status update on the issue
